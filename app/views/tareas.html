1.- Lee el siguiente articulo http://martinfowler.com/articles/microservices.html y explica que significa "Microservices Architecture"

Es una práctica para programar una aplicación como un conjunto de pequeños "servicios". Normalmente se genera una aplicación en un mismo lenguaje que corre desde un mismo servidor y que se divide en pequeñas partes pero al estar dependiendo una de la otra al momento que se haga un cambio se tiene que reestructurar toda la aplicación. Con "microservices" esto no pasa, pues se programan en pequeños servicios que funcionan de manera intependiente, pero que a la vez forman una aplicación. Por cada servicio existe una base de datos, aunque puede ser que dos serivicios compartan una base de datos; esto es con el fin de que si se hacen cambios no se tenga que cambiar todo el programa, sino que los cambios no afectarán de forma negativa a los otros servicios que se tengan.

4. Explica como funcionan los routes en angular y pon codigo de ejemplo.

El Routing en AngularJS es lo que hace la pagina una Single Page Application (SPA), esto es que no tienes que hacer reload de otra pagina para poder ir a otra vista, simplemente lo guarda una vez al cargar toda la pagina web. Para hacer esto es necesario usar ngRoute, he aqui un ejemplo:

<!DOCTYPE html> <html> <script src="http://ajax.googleapis.com/ajax/libs/angularjs/1.4.8/angular.min.js"></script> <script src="//ajax.googleapis.com/ajax/libs/angularjs/1.4.8/angular-route.js"></script> <body ng-app="myApp"> <p><a href="#/">Main</a></p> <a href="#red">Red</a> <a href="#green">Green</a> <a href="#blue">Blue</a> <div ng-view></div> <script> var app = angular.module("myApp", ["ngRoute"]); app.config(function($routeProvider) { $routeProvider .when("/", { templateUrl : "main.htm" }) .when("/red", { templateUrl : "red.htm" }) .when("/green", { templateUrl : "green.htm" }) .when("/blue", { templateUrl : "blue.htm" }); }); </script> <p>Click on the links to navigate to "red.htm", "green.htm", "blue.htm", or back to "main.htm"</p> </body> </html> Se necesita el module de AngularJS Route, que es el primer script del código; también se necesita la dependencia "ngRoute" para poder tener acceso al $routeProvider y con éste se configuran las diferentes rutas.

Para poder tener diferentes access keys, es cuestión de escribirlas en el documento "credentials". Si vienen tenemos actualmente:

[default]

aws_access_key_id=ASDIHFAFNAOFNIGNFAK

aws_secret_access_key=aIJDAMDNFfsfDANFIKNAEdadADadD


Debajo de esta parte podemos agregar otro access key:

[default]

aws_access_key_id=ASDIHFAFNAOFNIGNFAK

aws_secret_access_key=aIJDAMDNFfsfDANFIKNAEdadADadD


[secondKey]

aws_access_key_id=FHIAFHIKAFNIKGNISSN

aws_secret_access_key=asiISFJIJFafSAFINHFafdfINSFNdA

Para poder usar una o la otra se debe especificar el perfil que si quiere utilizar:

aws s3 sync --acl=public-read --profile secondKey . s3://ruthbelinda
